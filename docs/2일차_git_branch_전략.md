# 🌿 5조 Git 브랜치 전략 보고서 (Stage 2)

본 문서는 **AI 헬스케어 5기 웹 개발 트랙 과제 프로젝트(흉부 X-Ray 기반 폐렴 환자 관리 백오피스)**의 효율적인 협업과 안정적인 소스코드 관리를 위해 선정된 Git 브랜치 전략을 정의합니다.

---

## 1. 브랜치 전략 비교 분석

### 1) Git Flow
- **개념**: 5가지 브랜치(`main`, `develop`, `feature`, `release`, `hotfix`)를 명확히 구분하여 제품의 출시 버전을 엄격하게 관리하는 전통적인 전략입니다.
- **장점**: 각 개발 단계(알파, 베타, 운영)가 분리되어 대규모 협업 시 코드 안정성이 극도로 높고 역추적이 용이합니다.
- **단점**: 구조가 다소 복잡하여 브랜치 관리에 많은 리소스가 소모되며, 빈번한 병합 충돌(Merge Conflict)이 발생할 수 있어 빠른 배포에는 불리합니다.

### 2) GitHub Flow
- **개념**: 하나의 핵심 브랜치(`main`)와 기능 브랜치(`feature`)만을 사용하여 수시로 배포 가능한 환경을 유지하는 단순하고 직관적인 전략입니다.
- **장점**: 구조가 단순해 진입 장벽이 낮고, Pull Request(PR) 중심의 빠른 코드 리뷰와 피드백, 그리고 지속적 배포(CD)에 최적화되어 있습니다.
- **단점**: 완충지대 없이 바로 운영 브랜치로 머지되므로, 철저한 자동화 테스트와 리뷰 프로세스가 선행되지 않으면 배포 환경의 안정성이 저해될 수 있습니다.

---

## 2. 5조 선정 전략: "실용적 간소화 Git Flow (Lightweight Git Flow)"

우리 5조는 Git Flow의 **체계적인 안정성**과 GitHub Flow의 **민첩함**을 동시에 챙기기 위해, 기존 Git Flow를 프로젝트 규모에 맞게 간소화한 **`main` - `develop` - `issue`** 3단계 브랜치 전략을 사용합니다.

```text
[main] (최종 배포 및 제출용 브랜치)
  │
[develop] (개발 통합 및 테스트 브랜치)
  │
  ├─ [feature/#1-get-users] (팀원: 전체조회 개발) ──▶ (develop으로 PR/Merge)
  └─ [feature/#2-get-user]  (팀원: 단건조회 개발) ──▶ (develop으로 PR/Merge)
```

📌 주요 브랜치 정의 및 운용 규칙
| 브랜치명 | 대상 부모 브랜치 | 머지 대상 브랜치 | 설명 및 운용 규칙 |
| :--- | :--- | :--- | :--- |
| **`main`** | - | - | **최종 결과물 및 배포용 브랜치**. 프로젝트 종료 시점에 가장 안정적인 코드만 최종 배포 버전으로 머지합니다. |
| **`develop`** | `main` | `main` | **개발 기준 브랜치 (통합 본부)**. 팀원들의 모든 작업 결과물이 모이는 완충지대이며, 항시 실행 및 테스트가 가능한 상태를 유지합니다. |
| **`feature/`** <br>*(또는 `issue/`)* | `develop` | `develop` | **기능 개발용 독립 브랜치**. 각 팀원이 담당 API 및 기능을 독립적으로 개발하는 공간입니다. 작업 완료 시 `develop`을 대상으로 PR을 발송합니다. |


## 3. 우리 팀의 전략 채택 이유

1. **안정적인 협업 완충지대 확보 (`develop` 브랜치 활용)**
    - 개발 중인 미완성 코드나 테스트 단계의 소스코드가 최종 제출 브랜치인 `main`에 직접 병합되는 리스크를 원천 차단합니다.
2. **충돌 최소화와 책임 개발 체계 구축**
    - 각 조원이 독립적인 기능 브랜치(`feature/이름-기능명`)에서 작업함으로써 소스코드 병합 충돌을 방지하고 작업 단위를 명확히 분리합니다.
3. **자연스러운 코드 리뷰 및 품질 보증**
    - 작업 완료 후 `develop`으로 병합할 때 필수적으로 Pull Request(PR) 프로세스를 거치며, 팀 규칙에 따른 상호 코드 리뷰 및 피드백 문화를 장려합니다.
4. **실무형 협업 프로세스 경험**
    - 현업 IT 기업에서 가장 보편적으로 차용하는 브랜치 운용 방식을 몸소 경험함으로써 팀원 전원의 협업 역량과 Git 숙련도를 극대화합니다.
