### app/repositories/

데이터베이스에 직접 접근하는 코드를 모아두는 영역이다. API나 서비스 코드에서 SQL 쿼리 또는 ORM 조회 코드를 직접 작성하면 코드가 복잡해지므로, 조회/생성/수정/삭제 같은 DB 작업을 repository 계층으로 분리한다.

예를 들어 `UserRepository`는 사용자 조회, 사용자 생성, 이메일 중복 확인 같은 DB 작업을 담당할 수 있다.

### app/schemas/

API 요청과 응답의 데이터 형식을 정의하는 영역이다. 보통 Pydantic 모델을 작성한다.

예를 들어 회원 생성 요청에는 `name`, `email`, `password`가 필요하고, 회원 응답에는 `password`를 제외해야 한다. 이런 요청/응답 구조를 schema로 분리하면 API 문서와 데이터 검증이 명확해진다.
