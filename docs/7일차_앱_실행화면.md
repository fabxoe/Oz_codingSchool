# 7일차 앱 실행 화면 및 검증 결과

## 1. 개요
* **목표**: 6일차에 구현된 비동기 AI 폐렴 예측 API와 7일차 프론트엔드 연동 결과를 로컬 도커 환경에서 최종 검증하고 기록합니다.
* **검증 환경**: Docker Compose (`fastapi`, `mysql`, `redis`, `worker`)

---

## 2. 주요 기능 실행 화면

### 진료 상세 정보 및 X-Ray 이미지 조회
* **경로**: `/medical-records/{record_id}`
* **설명**: 등록된 환자의 진료 기록 상세 페이지에서 업로드된 X-Ray 흉부 이미지가 정상적으로 렌더링되는지 확인합니다.

![진료 상세 및 X-Ray 화면](https://prod-files-secure.s3.us-west-2.amazonaws.com/.../example_image_url.png)

---

## 3. AI 폐렴 예측 결과 검증

### 동작 흐름
1. **예측 요청 (POST)**: 진료 상세 페이지에서 "AI 예측 결과보기" 버튼을 클릭하면 백그라운드 워커(`worker`)로 작업이 전달됩니다.
2. **폴링 상태 전이 (GET)**: 프론트엔드(`static/apis.js`, `pages.js`)가 폴링 방식을 통해 90초 이내에 작업 상태를 주기적으로 확인합니다.
3. **결과 표시**: AI 추론이 완료되면 신뢰도(Confidence)가 소수점 형태에서 백분율(%)로 변환되어 화면에 출력됩니다.

### 확인된 결과 항목
* **예측 상태**: 완료 (Negative / 정상 또는 Pneumonia 예측 결과)
* **신뢰도 (Confidence)**: 퍼센트(% 단위 환산 표시)
* **모델 버전**: `pneumonia_ensemble_v1`

---

## 4. 특이사항 및 팀 주의사항
* **비동기 처리**: CPU 추론 소요 시간(약 3~5초)을 고려하여 동기 응답 대신 비동기 큐(`Redis`) 및 폴링 방식을 채택하여 요구사항을 충족했습니다.
* **모델 파일**: 609MB 규모의 가중치 파일은 Git에 포함되지 않으므로, `worker/models/pneumonia_ensemble_v1/` 경로에 수동 배치 후 컨테이너를 실행해야 합니다.
